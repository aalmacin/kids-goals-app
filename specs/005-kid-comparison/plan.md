# Implementation Plan: Kid Comparison

**Branch**: `005-kid-comparison` | **Date**: 2026-05-03 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/005-kid-comparison/spec.md`

## Summary

Add a comparison page for kids that shows a family leaderboard (ranked by total points), daily chore completion progress, and weekly points summary. All data lives in existing tables (`kids`, `day_records`, `chore_completions`, `activity_log`). The primary work is new RLS policies for cross-sibling reads, new DB query helpers, a new `/compare` route, and new UI components.

## Technical Context

**Language/Version**: TypeScript (strict), Next.js 16.2.4
**Primary Dependencies**: Next.js (App Router, Server Components), Supabase, shadcn/ui, TanStack Query/Store
**Storage**: Supabase (PostgreSQL with RLS)
**Testing**: Playwright (E2E), Vitest (unit/integration)
**Target Platform**: Web (PWA)
**Project Type**: Web application (Next.js App Router)
**Performance Goals**: Page load < 3s; comparison data fetched in a single pass
**Constraints**: PWA-capable, nonce-based CSP, RLS must be enabled on every table
**Scale/Scope**: Single family app, ~10 screens, typically 2–5 kids per family
